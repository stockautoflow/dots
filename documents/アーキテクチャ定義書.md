# スマートドッツ（Smart Dots）

## アーキテクチャ定義書（Vanilla JS特化型）

### 1. 構成方針（Architecture Principles）

本アプリは、外部ライブラリやフレームワークを一切使用しない**Vanilla JS（生JavaScript）**、**HTML5 Canvas API**、および **Web Audio API** を用いて構築する。
「状態（データ）」と「描画（ビュー）」を厳格に分離する **ローカル状態駆動型アーキテクチャ（Local State-Driven Architecture）** を採用し、コードの肥大化を防ぎつつ、0歳児の高速連打に耐えうる堅牢性とミリ秒単位のパフォーマンスを保証する。

---

### 2. ディレクトリ・ファイル構成

開発・メンテナンス性を高めるため、モジュールごとにファイルを分割する。コンパイル環境（Node.js等）を不要とするため、ES Modules（`import/export`）を利用してブラウザに直接読み込ませる。

```
smart-dots/
│
├── index.html          # 単一のインプット・表示先（SPA基盤）
│
├── css/
│   └── style.css       # スタイリング、アニメーション、touch-action制御
│
└── js/
    ├── app.js          # エントリーポイント（初期化・画面遷移全体の統括）
    ├── state.js        # 状態管理（クリア日数、設定、LocalStorage同期）
    ├── curriculum.js   # 算術ロジック（日別の数・数式の動的生成）
    ├── render.js       # 描画エンジン（Canvasへのドッツ・数字・数式描画）
    └── audio.js        # 音声エンジン（Web Audio APIによるメモリ再生）

```

---

### 3. モジュール（レイヤー）間連携定義

各モジュールの責務を分離し、単方向のデータフロー（State ➔ Render/Audio）を徹底する。

```
      [ 1. User Interaction / Auto Timer ]
                       │
                       ▼
            [ 2. js/app.js (Controller) ]
                       │
        ┌──────────────┴──────────────┐
        ▼                             ▼
 [ 3. js/state.js ] ──────────> [ 4. js/curriculum.js ]
   (状態の変更・保持)               (本日の数・数式の算出)
        │                             │
        └──────────────┬──────────────┘
                       ▼
        ┌──────────────┴──────────────┐
        ▼                             ▼
 [ 5. js/render.js ]           [ 6. js/audio.js ]
   (Canvasへの高速描画)          (AudioBufferでの遅延ゼロ再生)

```

#### 3.1 各モジュールの詳細責務

1. **`state.js`（データストア）**
* アプリ内の全状態（現在の累計日齢、現在の画面、音量設定など）を一つのオブジェクト（`AppState`）として集中管理する。
* 状態が変更された際、自動的に `LocalStorage` へ書き込みを行う。


2. **`curriculum.js`（ロジックエンジン）**
* 『内容定義書』で定めた30日の基本数量マスターを持つ。
* `state` から日齢を受け取り、その日の「フラッシュ用配列」「クイズの正解・不正解」「数式のパーツ」を計算して返す純粋関数（Pure Function）群。


3. **`render.js`（ビュー・描画）**
* `index.html` 上の `<canvas>` 要素のみを操作対象とする。
* 重なり防止アルゴリズムを用いてドッツの円を Canvas に直接描画する。


4. **`audio.js`（オーディオエンジン）**
* アプリ起動時に `AudioContext` を生成し、1〜50の音声ファイルを `fetch` してメモリ（`AudioBuffer`）にロード完了させておく。
* 再生命令を受けた際は、通信を発生させずメモリから即時（遅延 0ms）再生する。

### 4. コアロジックの技術仕様

#### 4.1 Web Audio API による遅延ゼロ再生（`audio.js`）

通常の `<audio>` タグは使用せず、以下のバッファ再生方式を採用する。

```javascript
// audio.js 内の実装イメージ
export class AudioEngine {
    constructor() {
        this.ctx = new (window.AudioContext || window.webkitAudioContext)();
        this.bufferCache = {};
    }

    // 起動時に全音声を先読み（プリロード）
    async preloadAllSounds(maxNum = 50) {
        for (let i = 1; i <= maxNum; i++) {
            const res = await fetch(`audio/ja/${i}.mp3`);
            const arrayBuffer = await res.arrayBuffer();
            this.bufferCache[i] = await this.ctx.decodeAudioData(arrayBuffer);
        }
    }

    // 再生（遅延なし）
    play(number) {
        const source = this.ctx.createBufferSource();
        source.buffer = this.bufferCache[number];
        source.connect(this.ctx.destination);
        source.start(0);
    }
}

```

#### 4.2 HTML5 Canvas による超高速描画（`render.js`）

DOM（HTMLのタグ）の増減によるブラウザの再レンダリング負荷（Reflow/Repaint）を避けるため、描画はすべて同一の Canvas 上で行う。

```javascript
// render.js 内の実装イメージ
export function drawDots(canvasId, number, skinType) {
    const canvas = document.getElementById(canvasId);
    const ctx = canvas.getContext('2d');
    ctx.clearRect(0, 0, canvas.width, canvas.height); // 画面クリア

    const dots = generateDotCoordinates(number, canvas.width, canvas.height);
    
    dots.forEach(dot => {
        ctx.beginPath();
        ctx.arc(dot.x, dot.y, 15, 0, Math.PI * 2);
        ctx.fillStyle = (skinType === 'blue_circle') ? '#3182ce' : '#ff6b6b';
        ctx.fill();
    });
}

```

### 5. 乳幼児（0歳児）の誤操作防止要件への技術的対策

1. **ブラウザジェスチャーの完全無効化（`style.css`）**
画面のあらゆる場所をバシバシ叩かれても、拡大ズームやスクロール、ページ引っ張り更新（Pull-to-refresh）が絶対に起きないよう、最外周の `body` および全画面要素に以下を強制する。
```css
body, .screen {
    touch-action: none;
    overflow: hidden;
    position: fixed;
    width: 100vw;
    height: 100vh;
}

```


2. **右クリック・長押しメニューの禁止（`app.js`）**
子どもが画面を触り続けた際にコンテキストメニュー（コピー等）が出るのを防ぐ。
```javascript
window.addEventListener('contextmenu', e => e.preventDefault());

```
