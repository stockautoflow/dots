# スマートドッツ（Smart Dots）

## 要件定義書

### 1. はじめに

#### 1.1 本ドキュメントの目的

本ドキュメントは、乳幼児向け右脳開発Webアプリ『スマートドッツ』の開発における要件定義書である。本作における機能的・非機能的な要求仕様を明確にし、以降の基本設計および実装フェーズの基準とする。

#### 1.2 背景と狙い

従来の物理的なドッツカードセットは「重い」「めくる技術が必要」「カードの順序が混ざる」といった親の負担が大きかった。これらをWebアプリケーション化し、2025年最新の七田式思想（30日×3サイクル）に基づき自動化・ゲーム化することで、親の負担を最小限に抑えつつ、0歳児から取り組める継続性の高い教育ツールを提供する。

---

### 2. システム概要

本システムは、インストール不要でスマートフォンやタブレット（推奨環境）から手軽にアクセスできる「シングルページWebアプリケーション（SPA）」として構築する。

* **ユーザー（子ども）**：全画面で表示されるドッツを閲覧、およびクイズに対する画面タップを行う。
* **オペレーター（保護者）**：設定の変更、カリキュラムの開始、進捗カレンダーの確認を行う。

---

### 3. 機能要件

機能要件は「①子ども向け画面（フロント）」「②保護者向け画面（管理・設定）」「③システム共通機能」の3つに大別される。

#### ① 子ども向け画面（レッスンモード）

| 機能ID | 機能名 | 機能概要 |
| --- | --- | --- |
| **F-01** | 高速フラッシュ表示 | 指定された秒数（0.6秒/枚）でドッツカードを自動で切り替え表示する。画像遅延を起こさない描画ロジックとすること。 |
| **F-02** | 音声自動読み上げ | カードの切り替えと完全に同期し、その数字の日本語ネイティブ音声を再生する。 |
| **F-03** | 0歳児向け2択クイズ | 画面を左右に2分割し、出題に対して子どもが叩いた（タップした）領域を判定する。小さなボタンではなく、画面の左半分・右半分すべてをタップ判定エリアとする。 |
| **F-04** | クイズ合否演出 | ・正解時：明るい効果音（ピンポーン）とともに星などのアニメーションを表示。<br>

<br>・不正解時：不快な音（ブブー等）は鳴らさず、正解の選択肢を優しく振動させて注視を促す。 |
| **F-05** | 数式アニメーション | 第3サイクル用。2つのドッツの塊が中央で「合体」して1つの大きな塊になるビジュアルエフェクト処理を行う。 |

#### ② 保護者向け画面（ダッシュボード・設定）

| 機能ID | 機能名 | 機能概要 |
| --- | --- | --- |
| **F-06** | デイリーレッスン導線 | ユーザーの進捗（現在何日目か）を自動判定し、「今日のレッスン（Day X）」を1タップで開始できるボタンを配置。 |
| **F-07** | 進捗カレンダー | レッスンを完了した日に「スタンプ」または「色」がつく月カレンダー。親のモチベーション維持を図る。 |
| **F-08** | レッスンカスタム設定 | フラッシュ速度（0.5秒 / 0.8秒 / 1.2秒）、音声のON/OFF、ドッツスキンの固定化などを設定できる（※91日目以降にフル解放）。 |

#### ③ システム共通機能

| 機能ID | 機能名 | 機能概要 |
| --- | --- | --- |
| **F-09** | チャイルドロック | レッスン中、子どもが誤って「中断」や「画面操作」をしないためのロック機能。画面端のボタンを**2秒以上長押し**した際のみ、大人向けの算数クイズ（例: 4+3=?）を表示し、正解時のみホームに戻る。 |
| **F-10** | ローカル進捗保存 | ログイン不要の手軽さを担保するため、ユーザーの現在の日齢（1〜90日目）およびカレンダーの履歴データをブラウザ（LocalStorage）に保存する。 |

### 4. 非機能要件

| 項目 | 要求仕様 | 備考 |
| --- | --- | --- |
| **対象デバイス** | タブレット（iPad、Androidタブレット）、スマートフォン（iPhone、Android） | 画面が大きく見やすいタブレット環境を推奨とする。 |
| **対応ブラウザ** | Google Chrome, Safari, Microsoft Edge (いずれも最新版) | モバイルブラウザでの動作を必須とする。 |
| **UI/UX（視覚設計）** | レッスン中、広告やメニューバーなどの余計な要素は一切非表示とする（子どもの視線逸脱を防ぐため）。 | 極限までシンプルなホワイトバックを基本とする。 |
| **パフォーマンス** | フラッシュ切り替え時の音声遅延は **50ms以内** とする。 | 音声と視覚のズレは知育効果を著しく下げるため、事前のオーディオプリロードを必須とする。 |
| **操作性** | 画面のピンチイン・ピンチアウト（拡大縮小）を禁止する。 | 子どもの連打による意図しない画面拡大を防ぐため。 |
| **セキュリティ** | 個人情報は一切収集しない。 | LocalStorage内での完結、または匿名IDでの進捗管理に留める。 |

### 5. 画面一覧・遷移定義

Webアプリは、以下の4つの主要画面（コンポーネント）で構成される。

```
[ホーム画面（保護者向け）]
       │
       ▼ （「今日のレッスン」タップ）
[カウントダウン画面（3, 2, 1）]
       │
       ▼ （自動遷移）
[レッスン画面（子ども向け）] ── (中断/長押し) ──> [チャイルドロック画面]
       │                                                    │ (正解)
       ▼ （全カード終了で自動遷移）                         ▼
[終了演出画面（よくできました！）] ─────────────────> [ホーム画面に戻る]

```

### 6. スケジュール・今後の拡張性（フェーズ2）

本要件定義書は、コアとなる機能に絞った「MVP（最小限の製品）」を対象とする。フェーズ2以降において、以下の機能を拡張できるデータ設計を担保すること。

* 複数子どものプロフィール切り替え機能
* クラウドへのデータバックアップ機能（ユーザーログイン対応）
* 51〜100までの「レベル2」カリキュラムの追加
